Overview

After configuring and logging to ProcessMaker, this page explain how you can interact with, and use the case list in the ProcessMaker Mobile app, as well as go over several other options that are available when running a case in the app.

To know the features in this page when running a case, see Run a Case.

Start a New Case

The ProcessMaker Mobile app allows you to create a case of a process previously created in the ProcessMaker web edition at any time. The New Case list displays all starting tasks that you have been previously assigned to participate in. To start a case, follow the next steps:

  1. Log on to ProcessMaker Mobile.

  2. Tap the + icon located at the bottom of the screen. This icon displays in every screen of the ProcessMaker Mobile interface except when running a case.

  3. All of your available starting tasks of processes are listed on the screen. Tap a specific task to start a case or search for a process and then choose the desired task.

    The Case Tabs displays the main case view by default where a Dynaform opens automatically for you to work on and to submit as you do it in the ProcessMaker Web edition.

Note: The Output Document, Input Document, and External Step features are not available for ProcessMaker Mobile as step: they will not open or create the case if any of those features are on the first task of the display list. If a process has a Dynaform, then any of the mentioned features follows as a step on the display list, the ProcessMaker Mobile version skips that part and continues with the process. These features will be implemented in future versions. The following alert message displays “You can’t do this task in the mobile edition. Please login using the web edition.”

Search Processes and Cases

The ProcessMaker Mobile app allows you to search processes when you are starting a case or using some tray in the cases list. To search for a process or a case, follow the next steps:

  1. Log on to ProcessMaker Mobile

  2. Start a new case or in the case list, go to a tray.

  3. Each tray screen and the New Case screen display the Search field located at the top side of the screen. Tap this field.

  4. Search data in the following scenarios:

    • Search by the process name or task name in case that you are starting a case. Type the text you want to search. The process tasks are automatically filtered.

    • Search cases by the process name or the task name by default in case that you are in the Case List. After typing the text you want to search, next to the Search field, the ellipsis icon displays to search by:
      • Name, which is the default search that combines the process name and the task name
      • Case Number
      • Send by
      • Delegate Date
      Tap the ellipsis, and select a criteria.

      After typing a criteria, cases are automatically filtered.

Use the Case List

You can work on your cases directly from the ProcessMaker Mobile app through the cases list. In this section you will see how to use the case list, as well as several other options when running a case on ProcessMaker Mobile. To access the Case List, follow the next steps:

  1. Log on to ProcessMaker Mobile

  2. Tap the three horizontal lines icon or slide the screen to the right.

  3. To the left side of the screen, a menu displays the case list with a red circle indicating the current number of cases in each tray. Access a tray by tapping one.

The left menu displays the following trays:

  • Inbox
  • Unassigned
  • Participated
  • Draft

Each one is described in the following sections.

Inbox

The Inbox panel lists all the cases assigned to you. The cases are sorted first by their case number, and then by the name of the case. The list displays cases with one of the following tree case statuses: On time, At risk or Overdue. Cases that can be worked offline are marked with the label "Available Offline." To learn more about offline cases, see ProcessMaker Offline Mobile.

To start working on a case, tap the desired case or you can first search for a case and after that tap a case.

The Case Tabs displays the main case view by default where a Dynaform opens automatically for you to work on and to submit as you do it in the ProcessMaker Web edition.

To learn other options in the Case Tabs, see The Case Tabs.

Unassigned

The Unassigned panel displays all cases that can be claimed by a pool of users assigned to the task. You can assign yourself a task, which then removes it from the Unassigned panel. When there are cases in the Unassigned tray, then anyone can claim the case to work on it.

To start working on an unassigned case, tap the desired case or you can first search for a case and then tap the desired case.

The Case Tabs displays the main case view by default where a case summary displays details about the unassigned case, details like status, date, description and more. At the right-bottom of the screen, tap Claim This Case to assign the case to yourself and to start working on it.

To learn other options in the Case Tabs, see The Case Tabs.

Participated

The Participated panel lists all the cases that you have sent or participated in.

To view a participated case, tap the desired case or you can first search for a case and then tap the desired case.

The Case Tabs displays the main case view by default where the case summary displays details about the case like the status, date, description and more.

To learn other options, see The Case Tabs.

Draft

The Draft panel includes cases that are being created but have not been submitted to the next task yet. Cases are considered to be in Draft status after you edit the case data by submitting a Dynaform or after moving on to a subsequent step in the current task. The list displays cases with one of the following three case statuses: On time, At risk or Overdue.

To start working on a draft case, tap the desired case or you can first search for a case and then tap the desired case.

The Case Tabs displays the main case view by default where a Dynaform opens automatically for you to work on and to submit as you do it in the ProcessMaker Web edition.

To learn other options, see The Case Tabs.

The Case Tabs

When running a case, below the case title, navigation tabs display where you can easily access all information about the case like documents uploaded and generated, case history, and case notes. To view the Case Tabs, open a case from the case list or start a new case. The Case Tabs displays the following tabs:

  • The main case view
  • Uploaded and generated files
  • Case history
  • Case notes

The Main Case View

This panel displays information about the case to work on it. This tab could vary depending on which place it was accessed from. The behavior of this tab vary according to the following:

  • If the case has been accessed when creating a case, from the Inbox or the Draft tray, the main case view displays the case steps.

  • If the case has been accessed from the Participated tray, the main case view displays the case summary and the current task properties.

  • If the case has been accessed from the Unassigned tray, the main case view displays the case summary, the current task properties and also the Claim This Case button at the right-bottom of the screen.

Files Attached

The Files Attached panel lists documents uploaded and generated when the case was running in the ProcessMaker Web edition. In ProcessMaker Mobile, to see this option when running a case in the Case Tabs, tap the Clip tab.

The Files Attached panel has two subsections that can be stretched by taping the arrow icon located at the top-right of each section. One section lists the uploaded documents (File Uploaded) and the other section lists the generated documents (File Generated). You can download files with the relevant permission assigned.

Where each document displays the following information:

  • File name
  • Type: The title of the Input or Output Document associated to the file. Otherwise, if the file is not related to an Input or Output Document, it displays as an Attachment
  • Version: The latest file version
  • Created On: When the file was uploaded
  • Created By: The user who uploaded the file

The Case History

This panel displays the case history. To see this option when running a case, in the Case Tabs, tap the Clock with Arrow tab.

Each history record includes the task name, the status, the people assigned to the task, the date and the time when the status changed.

The Case Notes

This panel lists the case notes. To see this option when running a case, in the Case Tabs, tap the Bubble tab.

Each case note includes the note text, the people who wrote the note, the date and the time when the note was written.

If the case is in the Inbox, Draft tray or creating a case, and the case has the relevant permission, you can add a note by following the next steps:

  1. In the Case Note panel, tap the + icon located at the bottom-right side of the screen.

  2. The Add note window displays. Type your note.

  3. If you want to send an email notification, check Send mail.

  4. Tap Post to publish the note. Otherwise, tap Cancel to exit.

Additional Options

Besides the Case List, there are also a few more options to consider, such as:

  • Settings
  • Help
  • About
  • Logout

Tap an option.

Settings

The Settings option displays the endpoint configuration, where you can interact with the following options:

  • Server URL
  • Workspace
  • Clear Cache
  • Enable external libraries cache
  • Language
  • Synchronize cases and tables
  • About

Tap an option to edit a setting.

Server URL

As a part of the server configuration, in the current field, type the ProcessMaker URL server such as http://192.168.1.1 or http://pm.example.com. Then, tap OK.

Workspace

As part of the server configuration, in the Workspace field, type the workspace name such as workflow. If using a personal ProcessMaker server, the default workspace is workflow. Then, tap OK.

Clear Cache

ProcessMaker can clear the cache automatically in certain scenarios. This functionality improves the use of data and makes ProcessMaker Mobile faster when working with different requests to the server. It also allows the application to work more fluid after performing different tasks.

Clear Cache Functionality

Clear Cache only releases the cache memory. Remember that no files are deleted and this action performs the Clear Cache functionality automatically and periodically.

Note: When using Clear Cache functionality, the default Server URL is set up to Https. Since ProcessMaker Mobile App only works with Secure Domain Services, it is mandatory to use HTTPS URL.

This section lists the events in which ProcessMaker performs the clear cache functionality:

  • When opening the application.
  • When the application is inactive for five minutes in the background.

See the following sections regarding how to clear cache settings for each mobile operating system:

  • Clear Cache On Settings in the Android Mobile Phone
  • Clear Cache On Settings in the iOS Mobile Phone
Clear Cache On Settings in the Android Mobile Phone

Follow these steps to clear the application’s cache in the Android mobile phone:

  1. Enter the ProcessMaker Mobile Settings.

  2. Go to the Clear Cache section located below the Server configuration section.

    Notice the Auto clear cache button in the Clear Cache section which you can use everytime you want to clear the cache automatically when the app is opened or when the app is inactive for five (5) minutes in the background.

  3. A confirmation dialog displays. To confirm the action, select the OK button.

  4. A progress dialog displays.

  5. The cache clears successfully. The option Press to Clear Cache is not available when there is no logged on user with the ProcessMaker Mobile application. Therefore, the cache memory is empty.

Clear Cache On Settings in the iOS Mobile Phone

Follow these steps to clear the application’s cache in the iOS mobile phone:

  1. Enter the ProcessMaker Mobile Settings.

  2. Go to the Clear Cache section located below the Server configuration section.

    Notice the Auto clear cache button in the Clear Cache section which you can use everytime you want to clear the cache automatically when the app is opened or when the app is inactive for five (5) minutes in the background.

  3. On the Press to Clear Cache option, select the row.

  4. A confirmation dialog displays. To confirm the action, select the OK button.

  5. A progress dialog displays.

  6. The cache clears successfully. The option Press to Clear Cache is not available when there is no logged on user with the ProcessMaker Mobile application. Therefore, the cache memory is empty.

Enable External Libraries Cache

The Enable external libraries cache option is available to improve the performance of the ProcessMaker app. If this option is turned on, the JavaScript libraries are downloaded in the device cache to load Dynaforms and sub-forms faster.

The first time a Dynaform or a sub-form that contains external libraries is loaded, all these libraries are downloaded at the same time and stored in the device cache. Therefore, the second time the Dynaform or sub-form is loaded, the loading time is faster.

These libraries can be cleared from the device by going to Settings and tapping the Clear Cache option.

Language

This feature allows you change the language of the mobile app from English to another language.

Change the language by tapping the English option below the Language label.

A checkbox list displays with the available languages. For example:

  • English (Default)
  • Spanish
  • French
  • Portuguese
  • Arabic

Tap an option to change the language. For example, choose "Portuguese" and automatically the changes take effect throughout the whole application.

Note: To translate correctly the Dynaforms control labels, follow the next instructions in ProcessMaker Desktop:

  1. Upload the Dynaform translation file.
  2. Upload the ProcessMaker translation file.

Mobile Right-to-Left Language Support

The ProcessMaker Mobile app supports right-to-left function for the Arabic language.

To see this feature, select the Arabic language under Settings > Language.

The interface of the ProcessMaker Mobile app changes to support right-to-left orientation.

Error and alert messages also display from right to left.

You also are able to type text from right to left in the login interface.

After logging to the ProcessMaker Mobile app, you see that your Inbox tray looks like the image below:

The navigation bar displays at the right side of the screen.

When running a case, the Dynaform interface displays the text from right to left.

Note: To translate correctly the Dynaforms control labels, follow the next instructions in ProcessMaker Desktop:

  1. Upload the Dynaform translation file.
  2. Upload the ProcessMaker translation file.

If mobile controls are included in the Dynaform, their interfaces also change to support right-to-left orientation.

Synchronize Cases and Tables

This option downloads all cases that can be worked offline to your device. This option also downloads offline PM Tables. This option is available if tasks enable the offline option. To learn more about offline cases and PM Tables, see ProcessMaker Offline Mobile.

About

This option displays the installed ProcessMaker Mobile version and build in your device. The About information also displays when the ProcessMaker Mobile is without a logged user.

Help

This option redirects you to the ProcessMaker wiki. Useful information about the mobile application can be found on this page. Tap Help, and the mobile web browser opens the wiki page.

About

This option displays the installed ProcessMaker Mobile version and build in your device. Go to the main menu by sliding the screen to the right. Tap the About option.

A window displays the version of the app and the build code. Tap Ok to quit the window and to go back to the main screen.

Logout

Go to the main menu by sliding the screen to the right. Tap the Logout option. The Logout option asks Are you sure you want to logout? Any unsaved data will be lost. Tapping Logout quits the ProcessMaker app and go back to the main screen. Otherwise tap Cancel.
