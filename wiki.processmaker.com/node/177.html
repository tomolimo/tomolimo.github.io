ProcessMaker v. 2.0.44 (Release date: October 9th, 2012)


ProcessMaker has always been characterized to be one of the best open sources tool, not far from that release after release, new bugs were solved and new features came up to provide quality and to satisfy the needs of the community. For that reason and from version 2.0.44 ProcessMaker will be scheduled to be released every 3 months because this will help us to to keep a progressive and continuous improvement on ProcessMaker’s quality and to provide Community a better open source tool.

ProcessMaker 2.0.44 is published as Release Candidate with 190 bugs fixed and tested by QA. This version fixes critical issues with Alfresco triggers, case history and improvements with the cron executions and the overall performance.

ProcessMaker upgrade script has been improved to be executed 33% faster than before and by upgrading everything without repeating manually the process. Finally, a lot of new features has been included in this version. After a month of this release QA is going to report if the version can change its label to Stable based on the results of our high level test procedures.

Recommendations

Upgrading a new version

When a new version of ProcessMaker is updated it’s recommended to clean the cache to assure the correct apply of the new functionality. The clean cache option is available in Configuration / settings / clean cache option.

Upgrading a new language

When a new language is imported, it’s is strongly recommended to clean and build cache to assure labels to be applied correctly. The build cache option is available in Configuration / settings/ Cases Cache List Builder
